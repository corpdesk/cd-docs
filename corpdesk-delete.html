Understanding the Delete Operation in Corpdesk Modules

The Delete operation removes records from the database. Below is an explanation of how it works, using the CdGeoLocationController, its service, and the model.

Step 1: The Controller - Entry Point for Delete Requests

The Delete() method in the CdGeoLocationController handles delete requests and forwards them to the CdGeoLocationService. 


                      async Delete(req, res) {
                          try {
                              await this.svCdGeoLocation.delete(req, res);
                          } catch (e) {
                              await this.b.serviceErr(req, res, e, 'CdGeoLocationController:Delete');
                          }
                      }
Step 2: The Service - Business Logic Layer

The delete method in the CdGeoLocationService:

  • Extracts the deletion criteria from the request payload.
  • Prepares a serviceInput object for the deletion.
  • Calls BaseService.delete() to execute the operation. 

                      async delete(req, res) {
                          const q = this.b.getQuery(req);
                          const serviceInput = {
                              serviceModel: CdGeoLocationModel,
                              cmd: { action: 'delete', query: q }
                          };
                          this.b.delete$(req, res, serviceInput)
                              .subscribe((result) => {
                                  this.b.respond(req, res, result);
                              });
                      }
Conclusion

The Delete operation simplifies the removal of records by leveraging the centralized logic in the BaseService.
