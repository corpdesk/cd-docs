Modules in Corpdesk: Directory Structure and Naming Conventions

Corpdesk is designed with a structured directory and file naming convention to ensure scalability, consistency, and ease of migration between systems. Understanding these conventions is essential for developers working with Corpdesk modules, which include controllers, models, and services.

Directory and File Naming Structure

The core directory structure for Corpdesk resides within the CdApi directory under the src folder. This directory is divided into two primary folders:

  • app: Contains application-specific modules.
  • sys: Contains system-level modules and utilities.

Each module within these directories maintains a consistent structure with folders for controllers, models, and services. Developers can add extra folders as necessary, but the core structure must remain intact.

File Naming Conventions

File naming conventions are crucial for ensuring consistency, especially when migrating Corpdesk installations between systems. Key points to note:

  • All file and directory names are in lowercase to avoid compatibility issues on case-sensitive file systems.
  • The naming convention for files reflects their purpose and type:
    • Controllers: File names follow the format {module-name}.controller.ts.
    • Models: File names follow the format {module-name}.model.ts.
    • Services: File names follow the format {module-name}.service.ts.

It is worth noting that all the file namings are mapped to specific entities in the module and are also reflected in the database as tables. For example, the cd-geo module has seven entities:

  • Root: cd-geo
  • Root Type: cd-geo-type
  • Location: cd-geo-location
  • Physical Type: cd-geo-physical-type
  • Political Type: cd-geo-political-type
  • Proximity: cd-geo-proximity
  • Track: cd-geo-track

The naming convention appends the utility name to the module name, ensuring a direct mapping between the module's functionality and its representation in the database tables. Each utility is represented by a table in the database, maintaining a consistent relationship across all module folders, including controllers, models, and services.

These conventions provide an avenue for consistency, interoperability, and integration with automated processes, including AI-assisted development. The alignment between the directory structure, file naming, and database schema is a critical factor in enabling streamlined and scalable development workflows.

Importance of Consistency

The structured directory and naming conventions serve several purposes:

  • Migratability: Ensures seamless movement of Corpdesk installations across environments with different case sensitivity in file systems.
  • Scalability: Allows easy addition of new modules and extensions without disrupting existing code.
  • Interoperability: The standardized structure and naming conventions enable seamless integration with tools, frameworks, and automation processes, making it easier for multiple teams or systems to work together efficiently.
  • Readability: Maintains clarity for developers navigating the project structure.
