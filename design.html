Aligning Design with SACCO Hierarchy

The design of ASDAP aligns with the hierarchical organization of SACCOs. At the base level, SACCOs are categorized by type, often based on their industry or mode of operation. To enhance administrative efficiency and advocacy, SACCOs are further grouped hierarchically according to geographical regions, spanning sub-regions within a country, national levels, continental organizations, and ultimately global bodies.

Hierarchical Privilege System(User Scopes)

ASDAP organizes its users, institutions, and statistical data to align with geographic hierarchies, ensuring coherence with SACCO structures. These are considered in the system under a term called User Scope. Below are some user scopes.

  • Regular User: user who has registered but is not affiliated to any SACCO in the system yet.
  • SACCO Affiliated User: Regular user can join a SACCO
  • SACCO Admin: SACCO affiliated user, depending on their level, can be promoted to admin with privileges aligned to that level.
  • SACCO Admin Scope: SACCO Admins can also be limited to geograpphical tags based on location and hierarchy.
Public Filter Utility(Data Access Control)

Data access can be controlled by User Scope or a utility called Public Filter. Administrators can block data using the following parameters:

  • Data Batch Reference
  • SACCO Type
  • Geographical Location
  • Geo-political Type
  • Date Label

If a batch of data becomes contested, it can be blocked using the Public Filter. Exemptions can be granted to specific users or groups if necessary.

Public Filter Workflow
Data Input

Data is entered into the system, tagged with batch reference and other parameters.

Filtering

Administrators apply filters based on SACCO type, geographic location, or date label.

Data Blocking

Filtered data is blocked from public and user access.

Exemptions

Specific users or groups are granted exemptions to access the blocked data.
